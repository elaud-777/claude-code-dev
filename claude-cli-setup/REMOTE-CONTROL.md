# Remote Control 상세 가이드

Claude Code의 "Remote Control"은 **로컬 PC에서 실행 중인 세션을 다른 기기(웹, 모바일)에서 원격으로 조종**하는 기능입니다. [REMOTE-WORK.md](REMOTE-WORK.md)에서 정리했듯 "하나의 원격 환경 + 여러 PC는 접속만" 목적에는 Claude Code on the web이 더 맞지만, 아래 상황에는 Remote Control이 유용합니다:

- 로컬 파일시스템/로컬 MCP 서버 접근이 필요한 작업을 외출 중에 모바일로 확인·승인하고 싶을 때
- 이미 로컬에서 돌리던 작업을 잠깐 다른 기기(폰, 다른 PC의 브라우저)에서 들여다보고 싶을 때

공식 문서: https://code.claude.com/docs/en/remote-control.md (v2.1.176 이상 권장)

## 요구사항

- Pro / Max / Team / Enterprise 구독 (API 키 단독 사용은 불가)
- Team/Enterprise는 **조직 Admin이 `claude.ai/admin-settings/claude-code`에서 Remote Control을 먼저 활성화**해야 함 (조직 전체 단위 설정, 팀별 개별 설정 불가)
- Zero Data Retention(ZDR) 정책을 쓰는 조직은 Remote Control을 켤 수 없음

## 활성화 절차

### 1. 로컬 PC에서 세션 시작

```bash
# 서버 모드 (여러 원격 세션을 받을 수 있음, 기본 --capacity로 동시 세션 수 제어)
claude remote-control

# 또는 이름을 지정한 대화형 세션으로 바로 시작
claude --remote-control "내 작업 세션"
```

이미 실행 중인 로컬 세션이 있다면, 그 세션 안에서 `/remote-control` 명령으로도 켤 수 있습니다.

추가 플래그:
- `--name`: 세션 이름 지정 (다른 기기에서 목록 볼 때 구분용)
- `--spawn`: 새 작업을 같은 디렉토리 또는 별도 worktree로 스폰할지 설정
- `--capacity`: 서버 모드에서 동시에 받을 원격 세션 수 (인터랙티브 모드는 프로세스당 1개, 서버 모드는 최대 32개까지 설정 가능)
- `--verbose`: 상세 로그

명령 실행 후 터미널에 **세션 URL**이 표시됩니다. **스페이스바를 누르면 QR 코드로 토글**되어 모바일에서 바로 스캔할 수 있습니다.

### 2. 다른 기기에서 접속

세 가지 방법 중 하나:
1. 표시된 세션 URL을 브라우저에서 그대로 열기
2. 모바일 앱(iOS/Android)에서 QR 코드 스캔
3. claude.ai/code 또는 모바일 앱의 세션 목록에서 해당 세션 선택 (연결된 세션은 목록에 초록색 상태점으로 표시됨)

## 보안 모델

| 항목 | 내용 |
|---|---|
| 접속 가능한 사람 | **동일한 Anthropic 계정으로 로그인한 경우만** — 다른 사람을 초대하는 기능은 없음 |
| 네트워크 방식 | **Relay 방식**: 로컬 PC는 Anthropic API로 outbound HTTPS 폴링만 하고, inbound 포트를 열지 않음 (즉, 로컬 PC를 인터넷에 직접 노출하지 않음) |
| 전송 보안 | TLS + 여러 개의 단기 자격증명(각각 독립적으로 만료) |
| 세션 만료 | 네트워크가 약 **10분 이상** 끊기면 타임아웃 (재시작되면 세션이 다시 열릴 수 있음) |
| 강제 종료 | 로컬 프로세스를 종료하거나, 원격 기기 쪽에서 연결 끊기 |
| 저장되는 데이터 | 세션 기록(메시지/응답/도구 활동 로그)은 **Anthropic 서버에 저장**됨. 실제 명령 실행과 파일 접근 자체는 로컬 PC에 남음 |

## 권한(permission) 프롬프트 처리

- **모바일/웹에서는 권한 승인을 직접 할 수 없습니다.** 파일 편집이나 명령 실행 승인이 필요한 시점에 도달하면, `/config`에서 설정한 푸시 알림("actions required")으로 통지만 받습니다.
- ⚠️ **불확실한 부분**: 공식 문서에 원격에서 권한 요청이 발생했을 때 정확히 어떤 흐름으로 처리되는지(로컬 PC로 돌아가서 승인해야만 하는지, 다른 우회 경로가 있는지) 명시돼 있지 않습니다. 로컬 PC에서만 최종 승인이 가능한 것으로 보이나, 원격 조종의 실효성을 판단할 때 이 부분을 직접 테스트해보고 확인하는 걸 권장합니다.

## Team/Enterprise 관리자 설정

- Admin이 `claude.ai/admin-settings/claude-code`에서 Remote Control을 켜고 끔 (조직 전체 적용, 팀 단위 설정 없음)
- 개별 사용자는 자신의 설정에서 `disableRemoteControl`로 스스로 끌 수 있음
- **Trusted Devices(베타)**: 같은 관리자 페이지에서 활성화. 기기를 미리 등록해야 하고, 18시간마다 생체 인증(biometric) 재인증이 필요함

## 한계점

- **로컬 프로세스가 항상 켜져 있어야 함** — 터미널/PC를 끄면 세션이 즉시 종료됨 (이 점이 REMOTE-WORK.md에서 "완전한 원격 환경이 아니다"라고 판단한 핵심 이유)
- 네트워크가 10분 이상 끊기면 타임아웃
- **절전모드/화면잠금 상태에서도 프로세스가 살아있으면 계속 동작** — PC 화면을 잠가둬도 괜찝지만, PC 자체가 완전히 절전(sleep)에 들어가 프로세스가 일시정지되면 그 시점부터는 응답 불가
- 인터랙티브 모드는 프로세스당 원격 세션 1개, 서버 모드는 최대 32개(`--capacity`로 조절)
- **Ultraplan을 시작하면 기존 Remote Control 연결이 자동으로 끊깁니다**
- 동일 세션에 여러 기기가 동시 접속 가능한지는 공식 문서에 명시되어 있지 않음

## 요약: 언제 이걸 쓰나

- ✅ 로컬 전용 도구(MCP, 파일시스템)를 계속 쓰면서, 진행 상황만 외부에서 확인/가끔 지시하고 싶을 때
- ❌ PC를 아예 켜두지 않고 여러 기기에서 완전히 독립적으로 작업을 이어가고 싶을 때 → 이건 [REMOTE-WORK.md](REMOTE-WORK.md)의 Claude Code on the web이 맞음
